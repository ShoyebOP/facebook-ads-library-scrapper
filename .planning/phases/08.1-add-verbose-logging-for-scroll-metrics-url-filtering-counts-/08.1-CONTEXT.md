# Phase 08.1: Add verbose logging for scroll metrics, URL filtering counts, and stopping reason diagnostics - Context

**Gathered:** 2026-07-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Add detailed diagnostic logging to help debug scraper behavior — log scroll metrics (scroll height, URLs found per scroll, filtered count, running unique total), URL filtering breakdown per scroll, and a full diagnostic snapshot when the scraper stops (reason, final counts, total time, scroll count).

</domain>

<decisions>
## Implementation Decisions

### Log Level for Metrics
- **D-01:** All scroll metrics and URL filtering counts at INFO level — visible by default, no need to enable DEBUG
- **D-02:** Per-scroll breakdown line showing: scroll height, URLs found this scroll, filtered duplicates, running unique total

### Scroll Metrics Detail
- **D-03:** Track scroll height (page scroll position after scroll)
- **D-04:** Track URLs found per scroll (new profile URIs intercepted from GraphQL responses)
- **D-05:** Track filtered count per scroll (duplicates removed)
- **D-06:** Track running total of unique URLs collected so far
- **D-07:** No scroll count tracking — existing logs already show this

### URL Filtering Breakdown
- **D-08:** Per-scroll line format: `Found: 12, Filtered: 3 (dups), Unique: 9, Total: 142`
- **D-09:** Filtered count includes the reason (duplicates) — keeps it concise

### Stopping Reason Format
- **D-10:** Full diagnostic snapshot when scraper stops — includes reason string, final stats, total time, scroll count
- **D-11:** Example format: `Stopped: max-urls reached (200/200). Total time: 3m 22s. Scrolls: 45. Unique URLs: 200.`

### Retry Logging
- **D-12:** Existing retry logging sufficient — no changes needed (WARN level in `errors.ts:101-106` and `webhook.ts:95-98`)

### the agent's Discretion
- Exact log message format and field names
- Whether to add a `--verbose` flag to toggle these metrics on/off
- How to handle scroll height when page doesn't scroll (error case)
- Whether to log DOM cleanup stats (memory management details)

</decisions>
