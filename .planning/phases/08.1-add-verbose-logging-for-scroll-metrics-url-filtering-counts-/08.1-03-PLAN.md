**Restructure heartbeat logic in `src/scraper.ts`** (gap 4):

The current heartbeat block (lines 161-174) is nested inside `if (targetSet.size === before)` — it only fires when no new URLs are found. This means the heartbeat never logs during productive scrolls. The else branch (lines 171-173) resets `noNewUrlsCount` and `lastLogTime` but never logs.

**Fix: Move heartbeat timer OUTSIDE the if/else branch, after the if/else block but before incrementalSaver.**

The new structure:
```
// After the if (targetSet.size === before) { ... } else { ... } block:

// Heartbeat: fire every 30s regardless of URL activity
const elapsed = Date.now() - lastLogTime;
if (elapsed >= 30000) {
    scrollLogger.info(
        {
            scrollHeight,
            urlsFound,
            filteredCount: filteredThisScroll,
            runningTotal: targetSet.size,
            deadScrolls: `${noNewUrlsCount}/${maxNoNewScrolls}`,
        },
        `[heartbeat] Scroll #${scrollCount}: found=${urlsFound}, filtered=${filteredThisScroll}, unique=${targetSet.size - before}, total=${targetSet.size}, height=${scrollHeight} — dead scrolls: ${noNewUrlsCount}/${maxNoNewScrolls}`,
    );
    lastLogTime = Date.now();
}
```

Key changes:
1. Heartbeat fires on a 30s timer regardless of whether URLs were found
2. Each beat logs per-beat metrics (not just a summary of dead scrolls)
3. Structured fields include: scrollHeight, urlsFound, filteredCount, runningTotal, deadScrolls
4. Message format shows all per-beat values inline for easy scanning

Keep `noNewUrlsCount` incrementing inside the `if (targetSet.size === before)` block (unchanged).
Keep `noNewUrlsCount = 0` reset in the else branch (unchanged).
The heartbeat timer reset (`lastLogTime = Date.now()`) happens ONLY when the heartbeat fires, not on every scroll — so it truly measures 30s of silence.
