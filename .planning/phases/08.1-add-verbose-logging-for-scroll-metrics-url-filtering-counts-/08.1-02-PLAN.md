**1. Add `filteredCount` field to `ScraperResult` in `src/types.ts`** (gap 5 prep):
- Add `filteredCount: number;` field to the ScraperResult interface after `maxUrls`

**2. Fix scrollHeight in `src/scraper.ts`** (gap 3):
- Line 141: Change `window.scrollY` to `document.body.scrollHeight`
- scrollY returns the current scroll position (how far down the page the viewport is). As more ads load and the page grows, scrollY increases — appearing cumulative. scrollHeight returns the total height of the document body, which is the correct per-scroll page height metric.

**3. Fix "found" field calculation in `src/scraper.ts`** (gap 1):
- The current code computes `urlsFound = targetSet.size - before` which only counts unique additions to the Set (deduplicates silently). The "found" field should show total unfiltered URLs intercepted by GraphQL this scroll, including duplicates.
- Reorder: compute `filteredThisScroll` BEFORE `urlsFound`
- Change: `const urlsFound = (targetSet.size - before) + filteredThisScroll`
- This gives total unfiltered = unique additions + duplicates filtered out

**4. Fix "unique" field in the log message** (gap 2):
- Line 158: The log message has `Unique: ${runningTotal}` which is the same as `Total: ${runningTotal}` (both show cumulative targetSet.size)
- Change the Unique part of the message to: `Unique: ${targetSet.size - before}` (per-scroll new unique additions only)
- Keep `Total: ${runningTotal}` unchanged (cumulative total)

**5. Return filteredCount in ScraperResult** (gap 5 prep):
- In the return statement (line 181), add `filteredCount: getFilteredCount()` to the returned object
