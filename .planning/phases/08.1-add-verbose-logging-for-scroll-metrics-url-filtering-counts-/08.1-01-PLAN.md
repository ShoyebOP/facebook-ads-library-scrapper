- Export `getFilteredCount(): number` — returns `filteredCountTotal` (cumulative)
- Export `resetFilteredCount(): void` — resets `filteredCountTotal = 0` (called at start of each scrape)
- Modify `page.on('response')` handler: before calling `extractProfileUrls`, record `const beforeSize = profileUrls.size`. After extraction, compute `const added = profileUrls.size - beforeSize`. Since we don't know total `page_profile_uri` occurrences in the JSON, we track filtered as "attempted minus added" by counting `page_profile_uri` occurrences during extraction. 

**Refined approach for filtered count:** Modify `extractProfileUrls` to return `{ added: number; attempted: number }` instead of void. The function already recurses through JSON. Change the signature so each call returns the count of `page_profile_uri` fields found vs actually added to the set. The top-level call aggregates: `attempted` = total `page_profile_uri` found in JSON, `added` = new URLs added to set. Filtered = `attempted - added`. Update `filteredCountTotal += (attempted - added)` in the response handler.
