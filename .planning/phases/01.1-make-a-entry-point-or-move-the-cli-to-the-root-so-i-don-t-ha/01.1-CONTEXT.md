<domain>
## Phase Boundary

CLI usability improvement — make the scraper runnable from project root without specifying `src/cli.ts`, add custom ad library URL per preset with CLI override, clean up old JavaScript files, and create comprehensive project documentation.

<decisions>
## Implementation Decisions

### Entry Point Location
- **D-01:** CLI invoked via `bin` field in package.json pointing to `src/cli.ts`
- **D-02:** Command name: `fb-ad-scraper` (for global/npx installation)
- **D-03:** Also add `npm start` script (`bun src/cli.ts`) for local development
- **D-04:** Run from project root: `bun start "query"` or `fb-ad-scraper "query"`

### Custom Ad Library URL
- **D-05:** Config field: `adLibraryUrl` in each preset (full URL with `{query}` placeholder)
- **D-06:** CLI flag: `--url` for per-run override
- **D-07:** Precedence: CLI flag overrides config preset
- **D-08:** Default: current hardcoded BASE_URL from scraper.js (fallback when neither config nor CLI flag set)
- **D-09:** Placeholder syntax: `{query}` — replaced with URL-encoded CLI argument at runtime
- **D-010:** URL format example: `https://www.facebook.com/ads/library/?active_status=active&ad_type=all&country=ALL&q={query}`

### Old File Cleanup
- **D-011:** Remove `scraper.js` only (keep example_page_code.html as reference)
- **D-012:** Update all references to scraper.js in package.json, .gitignore, and any docs

### README Content
- **D-013:** Full documentation: installation, usage, configuration, CLI flags, examples, license
- **D-014:** Include complete config.json example with all fields (presets, adLibraryUrl, webhook, proxy)
- **D-015:** Standard sections: title, description, installation, quick start, usage, configuration, CLI reference, examples, license

### the agent's Discretion
- Exact bin field format in package.json
- URL encoding implementation details (which Bun API to use)
- README markdown structure and formatting choices
- Whether to add config.json schema documentation in README or separate file

</decisions>
