**Step 3 — Implement --env-file flag (per CONFIG-04):**

In `src/cli.ts`, after argument parsing and validation (around line 95), add env file loading logic:

1. Check if `argv.envFile` is defined
2. If so, read the file at that path using `Bun.file()` and parse it as a `.env` format (KEY=VALUE lines, ignore comments starting with #, ignore empty lines)
3. For each parsed variable, set `process.env[key] = value`
4. Log that env file was loaded

The env file format is simple: each line is `KEY=VALUE`, lines starting with `#` are comments, empty lines are skipped. Values may be quoted (strip surrounding quotes). Do NOT use a library — this is a simple parser for a well-known format.

The env file loading must happen BEFORE `main(argv)` is called so that environment variables are available to the pipeline.

Also add the env file loading to `src/index.ts` as a safety net — if `argv.envFile` is passed, load it at the start of `main()` before config loading. This ensures env vars are available even if the CLI entry point is bypassed.
