<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config preset → webhook URL | User-provided URL in config.json — SSRF risk if URL points to internal services |
| Webhook response → process | External service response may contain malicious data — not parsed, only status code checked |
| Query string → webhook payload | User input flows into JSON payload — log injection risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Information Disclosure | notifyWebhook | medium | mitigate | Validate webhook URL is HTTPS; log only status code, not response body; proxy credentials already redacted by pino redact config |
| T-03-05 | Tampering | notifyWebhook payload | low | accept | Payload contains only non-sensitive metadata (query, file path, count); downstream reads JSON file directly |
| T-03-06 | Denial of Service | notifyWebhook retry | low | accept | p-retry limits attempts to 3 with exponential backoff; 10-second timeout prevents hung requests |
| T-03-07 | Elevation of Privilege | webhook error isolation | medium | mitigate | Outer try/catch in notifyWebhook ensures webhook failure never crashes scraper; file is saved before webhook per D-07 |
</threat_model>
