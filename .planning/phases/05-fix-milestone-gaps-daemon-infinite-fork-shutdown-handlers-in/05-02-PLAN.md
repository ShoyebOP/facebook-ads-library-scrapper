**Part B — Wire shutdown handlers after scraper starts (per D-05, D-06, D-07, D-08):**

In `src/index.ts`, AFTER the `runScraper` call returns `urls`, wire the shutdown handlers. Only wire when running as daemon (check `process.env.SCRAPER_DAEMON_CHILD === '1'`).

The challenge: `browser` is scoped inside `runScraper()` and not accessible in `main()`. The solution: pass a `cleanup` callback that captures the browser via closure. Since `runScraper` has a finally block that closes the browser, and shutdown handlers fire during execution, we need the browser reference.

**Approach:** Modify the architecture so `main()` can pass a `cleanup` callback to `runScraper`. Add an optional `onBrowserReady` callback to `ScraperOptions` that receives the browser reference when it launches. In `runScraper`, after `launchBrowser` succeeds, call `onBrowserReady(browser)` if provided. Then in `main()`, capture the browser in a local variable via this callback and use it in the shutdown handler cleanup.

Implementation steps:
1. In `src/types.ts`, add `onBrowserReady?: (browser: Browser) => void` to ScraperOptions (import Browser type from playwright-core)
2. In `src/scraper.ts`, after `browser = await launchBrowser(...)` (line 58), call `options.onBrowserReady?.(browser)` if the callback exists
3. In `src/index.ts`, declare `let browser: Browser | null = null` before the scraper options. Add `onBrowserReady: (b) => { browser = b; }` to ScraperOptions. After `runScraper` returns, wire shutdown handlers:
   ```
   if (process.env.SCRAPER_DAEMON_CHILD === '1') {
       const { setupDaemonShutdown } = await import('./daemon.js');
       setupDaemonShutdown({
           saveState: () => {
               saveUrlsToFile(outputFile, urls);
               logger.info(`Saved ${urls.size} URLs during shutdown`);
           },
           cleanup: async () => {
               if (browser) {
                    try { await browser.close(); } catch (err) { logger.error({ err }, 'Failed to close browser during shutdown'); }
               }
           },
           logger,
       });
   }
   ```

Do NOT remove the existing try/finally browser close in `runScraper` — it still handles normal completion. The shutdown handler only fires on SIGINT/SIGTERM during execution.
