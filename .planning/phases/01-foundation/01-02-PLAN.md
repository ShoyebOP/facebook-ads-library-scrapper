**Step 1: Implement src/cli.ts** (per CONFIG-04, D-09, D-10, D-11)

Create `src/cli.ts` as the CLI entry point:

1. Import yargs and hideBin helper:
   `import yargs from 'yargs'`
   `import { hideBin } from 'yargs/helpers'`
   Use `.js` extension in imports per Bun ESM requirement.

2. Define CLI options with yargs:
   - `--query` (string, required) — Search keyword for Facebook Ads (per D-09: flag, not positional)
   - `--preset` (string) — Configuration preset name (resolves callback URL)
   - `--headless` (boolean, default: false) — Run browser in headless mode
   - `--proxy` (string) — Proxy server URL (HTTP/SOCKS5)
   - `--max-urls` (number) — Maximum URLs to collect
   - `--max-no-new-scrolls` (number, default: 10) — Stop after N scrolls with no new results
   - `--daemon` (boolean, default: false) — Run as background process
   - `--callback` (string) — Webhook callback name (overrides preset)
   - `--env-file` (string) — Path to env file (per CONFIG-04)

3. Use `.strict()` to reject unknown arguments.

4. Implement validation:
   - If `--query` is missing, print `"Error: query required"` and `process.exit(1)` per D-11
   - Parse numbers with parseInt, handle NaN gracefully
   - Validate `--max-urls` and `--max-no-new-scrolls` are positive integers

5. After parsing, call `main(argv)` from `src/index.ts` with parsed arguments.

6. Handle top-level errors: catch and print `"Error: {message}"`, exit with code 1.

**Step 2: Implement src/index.ts** (pipeline wiring)

Update `src/index.ts` to be the pipeline orchestrator:

1. Import `loadConfig` and `resolvePreset` from `src/config.js`
2. Import stubs from `src/browser.js`, `src/scraper.js`, `src/output.js`
3. Export `async function main(argv: CliArgs)` that:
   a. Loads config via `loadConfig()`
   b. If `--preset` provided, resolves preset via `resolvePreset(config, presetName)`
   c. Logs configuration summary: query, maxUrls, outputFile
   d. Calls `launchBrowser()` from browser stub (returns placeholder for now)
   e. Calls `runScraper()` from scraper stub (returns placeholder for now)
   f. Calls `saveOutput()` from output stub (returns placeholder for now)
   g. Logs "Pipeline complete (stubs — real implementation in Phase 2)"

4. Define `CliArgs` type inline or export from cli.ts:
   ```typescript
   type CliArgs = {
     query: string;
     preset?: string;
     headless: boolean;
     proxy?: string;
     maxUrls?: number;
     maxNoNewScrolls: number;
     daemon: boolean;
     callback?: string;
     envFile?: string;
   }
   ```

```typescript
});
```
